## 核心模块

### 1. 做市商定价格模型

**核心思想**：价格不仅仅是成本加成，而是使**期望利润最大化**的结果。

**数学模型**：
```
max_P E[Π] = (P - C) · (1 - R(P)) · Q(P) - C_ret · R(P) · Q(P) - H(q)

最优价格：
P* = (|ε| / (|ε| - 1)) · (C + (R · C_ret) / (1 - R))
```

**关键洞察**：
- 退货率R是价格P的内生函数
- 最优价格自动包含风险溢价
- 价格弹性ε决定定价能力

### 2. 元学习冷启动（MAML）

**核心问题**：新SKU没有历史数据，如何定价？

**解决方案**：
1. 在大量历史SKU上预训练元模型
2. 面对新SKU时，仅需5-10个订单即可快速适应
3. 结合专家策略兜底

**效果**：冷启动期从50-100订单 → 5-10订单（缩短90%）

### 3. Greeks风险指标

**核心思想**：使用JAX自动微分实现精确的风险度量

**Greeks映射**：
| Greek | 金融含义 | 电商含义 |
|-------|---------|---------|
| Delta | 价格敏感度 | 利润对价格的一阶敏感度 |
| Gamma | 凸性 | 利润对价格的二阶敏感度 |
| Vega | 波动率敏感度 | 利润对退货率波动的敏感度 |
| Rho | 利率敏感度 | 利润对库存的敏感度 |

### 4. 分级定价引擎

**核心思想**：在延迟和精度之间取得平衡

| 层级 | 延迟 | 覆盖 | 技术 |
|------|------|------|------|
| Tier 1 | <10ms | 80% | L1 Cache |
| Tier 2 | <50ms | 15% | Redis + ONNX |
| Tier 3 | <200ms | 5% | Ray Serve |

**加权平均**：P99 < 25ms

## 数据流

```
定价请求
    │
    ▼
┌─────────────┐
│ 分级定价引擎 │
└─────────────┘
    │
    ├─── 热数据 ───→ L1 Cache ───→ 返回价格 (<10ms)
    │
    ├─── 温数据 ───→ Redis + ONNX ───→ 返回价格 (<50ms)
    │
    └─── 冷数据 ───→ Ray Serve DRL ───→ 返回价格 (<200ms)
```

## 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 自动微分 | JAX | 精确导数，GPU加速 |
| 深度学习 | PyTorch | 生态成熟 |
| 模型服务 | ONNX Runtime | 轻量级推理 |
| 分布式 | Ray | 训练与服务统一 |
| 缓存 | Redis | 高性能KV存储 |
| 数据库 | PostgreSQL + TimescaleDB | 时序数据支持 |
